# 产品需求文档 (PRD)：AcroSnap (Windows 多窗口沉浸式版本)

## 1. 项目概述 (Project Overview)
**AcroSnap** 是一款专为 CS 与 AI 领域研究人员设计的桌面端缩写词典与知识管理工具。本版本采用**多窗口隔离架构**，将“用完即走的极速查词”与“沉浸式的 AI 深度探索”在物理视窗上进行分离。应用常驻后台，通过不同的全局快捷键唤起对应的功能窗口，辅助研究人员在阅读文献或写代码时，实现零打断的知识获取与沉淀。

## 2. 核心目标 (Core Objectives)
* **场景隔离**：快捷查询不干扰主屏幕视野；深度问答提供足够大的画布。
* **极速响应**：毫秒级热启动与窗口唤起，本地缓存优先机制。
* **专业渲染**：针对 AI 领域的数学公式与代码片段，提供原生级的高清排版与渲染。
* **无缝串联**：窗口间支持数据传递（IPC），实现从“浅层查询”到“深度追问”的顺滑过渡。

---

## 3. 架构形态与多窗口定义 (Window Management)

应用采用“1个隐形主控进程 + 2个功能子窗口”的架构模型：

| 窗口标识 | 唤起方式 | UI 尺寸规范 | 核心职责 | 生命周期与焦点行为 |
| :--- | :--- | :--- | :--- | :--- |
| **主控进程 (Main)** | 随系统开机自启 | 0x0 (完全隐藏) | 维持应用生命周期，监听全局快捷键，管理托盘菜单，调度子窗口。 | 始终存活于后台，不可见。 |
| **查询窗口 (Search)** | `Alt + 1` | 宽 600px，高 60px (展开最大 400px) | 提供类似 Spotlight 的极简搜索框，快速匹配本地词条或触发 AI 简答。 | 无边框、居中、透明背景。**失去焦点时立即自动隐藏**。 |
| **问答窗口 (Chat)** | `Alt + 2` | 宽 800px，高 600px | 提供对话流界面，支持对特定术语进行多轮深度对话。 | 无边框、居中、带有自定义关闭/最小化按钮。**失去焦点时不隐藏**，需手动关闭。 |

## 4. 详细功能需求 (Functional Requirements)

### 4.1 极速查询流 (隶属 Search 窗口)
这是最高频的使用场景，要求绝对的克制与高效。

* **唤起与隐藏**：
    * 按下 `Alt+1` 瞬间屏幕居中弹出输入框，光标自动锁定。
    * 点击屏幕空白处（失去焦点）或按 `Esc` 键，窗口瞬间隐藏，同时清空刚才的输入内容。
* **模糊搜索与本地匹配**：
    * 用户键入字符时，实时过滤本地数据库（Supabase 拉取的缓存）。
    * 界面以列表形式向下展开候选词，支持键盘 `↑` `↓` 键选择，`Enter` 键确认。
* **AI 解析兜底机制**：
    * 若本地词库未命中，列表提示“未找到，按回车让 AI 解释”。
    * 触发后，界面展示骨架屏加载动画，随后渲染出 AI 返回的标准化卡片（全称、中文释义、核心公式）。
* **一键保存与流转**：
    * 卡片提供 `[保存至本地库]` 按钮：点击后静默同步至云端。
    * 卡片提供 `[深度追问]` 按钮：点击后，当前查询窗口自动隐藏，系统自动唤起 **Chat 窗口**，并将该缩写词条通过事件总线 (Event Bus) 发送给 Chat 窗口作为对话上下文。

### 4.2 沉浸式对话流 (隶属 Chat 窗口)
用于处理复杂的、需要结合上下文的学术概念解释。

* **唤起机制**：
    * 可通过 `Alt+2` 直接唤起空白对话窗口。
    * 可通过查询窗口的 `[深度追问]` 按钮被动唤起并携带初始话题。
* **对话交互**：
    * 提供标准的气泡式对话界面（类似主流 LLM 聊天界面）。
    * 支持 Markdown 语法与复杂的 KaTeX 公式渲染（必须保证多行公式对齐不溢出）。
* **知识提取**：
    * 当 AI 的回答中包含有价值的新缩写或定义时，界面提供 `[提取并入库]` 的悬浮按钮，支持通过结构化提示词再次调用 AI，将其格式化为标准的词典条目并存入 Supabase。

### 4.3 数据管理与云端同步
* **云端数据源**：统一采用云端数据库存储核心术语表。
* **启动拉取**：应用每次冷启动时，全量拉取云端数据至本地内存/IndexedDB 中，确保后续查询为 0 延迟。
* **静默增量同步**：在任何窗口产生的“保存”动作，均采用先更新本地 UI（乐观更新），后静默发起网络请求同步至云端的策略，不阻塞用户操作。

### 4.4 系统托盘与设置 (System Tray)
* 提供系统右下角托盘图标驻留。
* 右键菜单包含：
    * **设置 API Key**：弹窗输入大模型 API Key（本地加密存储）。
    * **强制同步**：手动触发本地与云端的数据对齐。
    * **退出应用**：彻底杀掉后台主进程与所有子窗口。

## 5. 非功能与体验需求 (Non-Functional Requirements)

* **视觉与 UI 规范**：
    * **去系统化**：彻底移除 Windows 默认的粗糙标题栏（Title Bar）。
    * **材质与色彩**：支持深浅双色模式跟随系统。窗口背景需使用亚克力（Acrylic）或云母（Mica）材质，提供微妙的半透明毛玻璃质感。
    * **动画体验**：窗口的弹出、隐藏，以及搜索列表的向下展开，需带有缓动动画（Spring/Ease-out），避免生硬的闪现。
* **性能约束**：
    * 待机状态下（窗口全隐藏），整体内存占用控制在 50MB 左右。
    * 从按下快捷键到窗口完全渲染并锁定焦点的延迟 < 100ms。
* **异常处理**：
    * 无网络状态下：应用需能正常唤起并查询本地已缓存词条，并友好提示“当前离线，AI 解析与同步功能不可用”。

## 6. 未来演进预留 (Future Proofing)

* **跨端架构准备**：在实现 UI 时，要求前端组件与业务 Hook（如查库逻辑、调接口逻辑）彻底解耦，不使用特定于桌面的 DOM 操作，为后期直接复用该部分逻辑并迁移至 Android 原生应用（如通过 Tauri v2 打包 APK）打好基础。
* **数据结构扩展**：词条数据库的表结构中，需预留 `tags`（标签数组）与 `related_links`（关联文献链接）字段，方便未来支持按学科分类检索。
